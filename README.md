# Requirement document
## Functional requirement
### 1.ระบบสามารถ บันทึกการเข้า–ออกของรถ ได้
### 2.ระบบสามารถ ตรวจสอบจำนวนที่จอดว่าง/เต็ม แบบเรียลไทม์
### 3.ระบบสามารถ คิดค่าบริการจอดรถตามเวลา ที่ใช้จริง
### 4.ระบบสามารถ รองรับบัตรจอดรถ / บัตรสมาชิก / QR Code
### 5.ระบบสามารถ เปิด–ปิดไม้กั้นอัตโนมัติ เมื่อรถเข้า–ออก
### 6.ระบบสามารถ ค้นหาข้อมูลรถจากทะเบียนหรือหมายเลขบัตร
### 7.ระบบสามารถ จัดการข้อมูลสมาชิก (เพิ่ม/ลบ/แก้ไข)
### 8.ระบบสามารถ รองรับการชำระเงินหลายรูปแบบ (เงินสด, บัตร, e-Payment)
### 9.ระบบสามารถ แสดงรายงานสถิติการใช้งานที่จอดรถ
### 10.ระบบสามารถ แจ้งเตือนเมื่อที่จอดรถเต็ม หรือใกล้เต็ม
## Non Functional requirement
### 1.ใช้งานง่ายไม่ยุ่งยาก
### 2.ui ดึงดูดน่าสนใจ
### 3.ระบบตอบสนองได้รวดเร็ว
### 4.ไม่เข้าถึงข้อมูลส่วนตัวมากเกินไป
### 5.ระบบมีความเสถียรไม่ล่มง่าย



# TOR ข้อกำหนดคุณสมบัติระบบบริหารจัดการที่จอดรถ
## 1. ระบบแสดงผลสำหรับผู้ใช้บริการ (Front-End)
### 1.1 ระบบต้องมีส่วนติดต่อผู้ใช้ (User Interface) ที่ใช้งานง่าย ทันสมัย และรองรับการใช้งานบนอุปกรณ์หลากหลายประเภท เช่น คอมพิวเตอร์ แท็บเล็ต และโทรศัพท์มือถือ (Responsive Design)
### 1.2 ระบบต้องสามารถแสดงสถานะที่จอดรถแบบเรียลไทม์ โดยแสดงจำนวนที่จอดว่าง จำนวนที่จอดเต็ม และพื้นที่จอดที่ใช้งานอยู่
### 1.3 ระบบต้องสามารถรองรับการใช้งานของผู้ใช้บริการโดยไม่จำกัดจำนวนผู้ใช้งานพร้อมกัน
### 1.4 ระบบต้องสามารถแสดงข้อมูลค่าบริการจอดรถ อัตราค่าจอด และเงื่อนไขต่าง ๆ ให้ผู้ใช้บริการรับทราบได้

## 2. ระบบบันทึกการเข้า–ออกของรถยนต์
### 2.1 ระบบต้องสามารถบันทึกข้อมูลการเข้าและออกของรถยนต์แต่ละคันโดยอัตโนมัติ
### 2.2 ข้อมูลที่ต้องบันทึกประกอบด้วย
 - หมายเลขทะเบียนรถ
 - วันที่และเวลาเข้า
 - วันที่และเวลาออก
 - หมายเลขบัตรจอดรถ / QR Code / บัตรสมาชิก
### 2.3 ระบบต้องสามารถเชื่อมต่อกับอุปกรณ์ไม้กั้นอัตโนมัติ (Barrier Gate) เพื่อควบคุมการเข้า–ออกของรถยนต์

## 3. ระบบรองรับบัตรจอดรถและ QR Code
### 3.1 ระบบต้องรองรับบัตรจอดรถทั่วไป (Parking Card)
### 3.2 ระบบต้องรองรับบัตรสมาชิกสำหรับผู้ใช้บริการประจำ
### 3.3 ระบบต้องรองรับการเข้า–ออกด้วย QR Code
### 3.4 ระบบต้องสามารถตรวจสอบความถูกต้องของบัตรหรือ QR Code ก่อนอนุญาตให้เปิดไม้กั้น

## 4. ระบบคำนวณค่าบริการจอดรถ
### 4.1 ระบบต้องสามารถคำนวณค่าบริการจอดรถตามระยะเวลาที่ใช้งานจริง
### 4.2 ระบบต้องสามารถกำหนดอัตราค่าจอดได้หลายรูปแบบ เช่น
 - คิดค่าบริการรายชั่วโมง
 - คิดค่าบริการแบบเหมาจ่าย
 - อัตราพิเศษสำหรับสมาชิก
### 4.3 ระบบต้องสามารถแสดงรายละเอียดค่าบริการก่อนการชำระเงินได้

## 5. ระบบการชำระเงิน
### 5.1 ระบบต้องรองรับการชำระเงินหลายรูปแบบ ได้แก่
 - เงินสด
 - บัตรเครดิต / บัตรเดบิต
 - ระบบ e-Payment
### 5.2 ระบบต้องสามารถเชื่อมต่อกับระบบชำระเงินภายนอกได้
### 5.3 หลังจากการชำระเงินสำเร็จ ระบบต้องสั่งการให้ไม้กั้นเปิดโดยอัตโนมัติ

## 6. ระบบค้นหาข้อมูล
### 6.1 ระบบต้องสามารถค้นหาข้อมูลรถยนต์จากหมายเลขทะเบียนรถ
### 6.2 ระบบต้องสามารถค้นหาข้อมูลจากหมายเลขบัตรจอดรถหรือหมายเลขสมาชิก
### 6.3 ระบบต้องสามารถแสดงประวัติการจอดรถย้อนหลังได้

## 7. ระบบจัดการสมาชิก (Back-End)
### 7.1 ระบบต้องมีระบบจัดการสมาชิกสำหรับผู้ดูแลระบบ
### 7.2 ผู้ดูแลระบบต้องสามารถ
 - เพิ่มข้อมูลสมาชิก
 - แก้ไขข้อมูลสมาชิก
 - ลบข้อมูลสมาชิก
### 7.3 ระบบต้องสามารถกำหนดสิทธิ์และประเภทของสมาชิกได้

## 8. ระบบรายงานและสถิติ
### 8.1 ระบบต้องสามารถแสดงรายงานสรุปการใช้งานที่จอดรถในรูปแบบกราฟและตาราง
### 8.2 รายงานต้องสามารถแยกตามช่วงเวลา เช่น
 - รายวัน
 - รายเดือน
 - รายปี
### 8.3 ระบบต้องสามารถแสดงสถิติต่าง ๆ เช่น
 - จำนวนรถเข้า–ออก
 - ระยะเวลาการจอดเฉลี่ย
 - รายได้จากค่าจอดรถ

## 9. ระบบแจ้งเตือน
### 9.1 ระบบต้องสามารถแจ้งเตือนเมื่อที่จอดรถเต็ม
### 9.2 ระบบต้องสามารถแจ้งเตือนเมื่อที่จอดรถใกล้เต็มตามค่าที่กำหนด
### 9.3 การแจ้งเตือนต้องแสดงผลผ่านหน้าจอระบบของผู้ดูแล

## 10. ข้อกำหนดคุณลักษณะด้าน Non-Functional Requirement
### 10.1 ระบบต้องใช้งานง่าย ไม่ซับซ้อน
### 10.2 ระบบต้องมีความรวดเร็วในการตอบสนอง
### 10.3 ระบบต้องมีความเสถียร สามารถทำงานต่อเนื่องได้
### 10.4 ระบบต้องคำนึงถึงความปลอดภัยของข้อมูล และไม่เข้าถึงข้อมูลส่วนตัวเกินความจำเป็น
### 10.5 ระบบต้องสามารถรองรับการขยายระบบในอนาคตได้
